Programación con ChatGPT


¿Hasta dónde podemos llegar?

Introducción

Propósito

Evaluar la capacidad de ChatGPT para producir código comparado con programadores humanos.

  • Enfoque
    • Habilidad de resolución.
    • Optimización: Tiempo de ejecución y uso de memoria.
  • Lenguajes estudiados
    • C++
    • Java

Investigaciones Previas

  • OpenAI (2022): IA como procesador de lenguaje natural y generador de código.
    • Generó la necesidad de analizar la capacidad de la IA de entender problemas en el campo del desarrollo de software.
  • Marr (2023) y Phillips (2022): ChatGPT como herramienta para mejorar los procesos de desarrollo de software.
    • Generó la necesidad de analizar la calidad del código producido por la IA.

Metodología

Preguntas de Investigación

  1. ¿Qué tan bien puede ChatGPT resolver un problema general de programación?
  2. ¿Cómo se comparan las soluciones obtenidas de ChatGPT en cuanto a tiempo de ejecución y uso de memoria contra las producidas por humanos?

Muestra

  • 120 problemas seleccionados aleatoriamente para cada lenguaje (C++ y Java) tomados de LeetCode.
    • 9 categorías: listas, cadenas, ordenamiento, matemáticas, tablas hash, programación dinámica, algoritmo voraz, pilas y colas.
    • 3 niveles de dificultad: fácil, media y difícil.

Proceso

  1. Se le pide a ChatGPT resolver un problema
  2. El código generado se sube a Leetcode.
  3. Leetcode evalúa si la solución es correcta y además verifica el tiempo de ejecución y el uso de memoria.
  4. Se impone un máximo de 3 intentos, pasado este límite el problema se marca como sin solución posible o muy demandante.

Resultados

Habilidad de Resolución

  • Resultados similares para las categorías Fácil y Media.
  • El desempeño decrece para la categoría Difícil y algunos problemas no pueden resolverse.

Tiempo de Ejecución

  • Clasificación promedio según Leetcode (en percentiles):
    • Fácil: 58.
    • Media: 59.
    • Difícil: 56.
  • Prometedor en categorías Fácil y Media.
  • Java presenta resultados inferiores en las categorías Media y Difícil.

Uso de Memoria

  • Clasificación promedio según Leetcode (en percentiles):
    • Fácil: 58.
    • Media: 47.
    • Difícil: 56.
  • Comparable con humanos en las categorías Fácil y Media.
  • Inferior en la categoría Difícil.

Discusión

Interpretación

  • Buen desempeño resolviendo problemas de dificultad Fácil y Media.
  • Dificultades para resolver problemas de la categoría Difícil.
  • Lejos de alcanzar la optimización lograda por los programadores humanos en cuanto a tiempo de ejecución y memoria más allá de obtener resultados prometedores en algunas situaciones.

Conclusión

ChatGPT genera código en C++ y Java de calidad inferior a la de programadores humanos, especialmente en tiempo de ejecución y uso de memoria.

Recomendaciones

  • Entrenamiento especializado para ChatGPT.
  • Incluir más lenguajes en futuros estudios.
  • Explorar nuevas métricas de calidad del código.
